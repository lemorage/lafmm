# LAFMM

**Livermore's Anticipating Future Movements Map** — an interactive terminal tool that implements Jesse Livermore's six-column price recording system from *How to Trade in Stocks* (1940).

You maintain CSV files with daily prices. The tool renders Livermore's 18-column map — column placement, pivotal points, trend confirmations, trading signals, Key Price, and market trend — all computed from the original rules.

Both human-readable (interactive TUI) and agent-readable (structured markdown). Designed for **Claude Code** (v2.1.59+ recommended).

## Quick Start

```bash
uv sync
uv run lafmm
```

First run creates `~/.lafmm/` with everything you need. Then:

```bash
cd ~/.lafmm && claude
```

Claude Code reads `CLAUDE.md` → `AGENT.md`, auto-discovers skills in `.claude/skills/`, and analyzes your market state. You add price data, the agent does the rest.

> Requires [Claude Code](https://claude.ai/download) v2.1.59+ for auto-memory. Skills, memory, and CLAUDE.md integration are built around Claude Code's native capabilities.

## How It Works

### For Humans

```bash
uv run lafmm       # launches interactive TUI
```

Navigate with arrow keys, `Enter` to drill in, `Esc` to go back, `q` to quit. Three levels: market dashboard → group (18-column map) → individual stock.

### For Agents

```bash
cd ~/.lafmm
cat cache/market.md               # market overview
cat cache/semis/group.md          # group detail with 18-col map
cat cache/semis/NVDA.md           # individual stock sheet
```

Structured markdown, self-contained per file. The agent reads `AGENT.md` for identity and domain knowledge, `cache/` for state, `skills/` for workflows.

## Data Layout

```
~/.lafmm/
├── AGENT.md                    # system prompt for any LLM agent
├── CLAUDE.md                   # @AGENT.md — bridges Claude Code
├── data/                       # the truth (TOML + CSV input)
│   ├── us-indices/             # scaffolded by default
│   │   ├── group.toml          # leaders = ["SPY", "QQQ"]
│   │   ├── SPY/2026.csv        # year-partitioned price data
│   │   ├── QQQ/2026.csv
│   │   ├── DIA/2026.csv
│   │   └── IWM/2026.csv
│   └── {group}/                # add sector groups here
│       ├── group.toml
│       └── {TICKER}/{YEAR}.csv
├── profile.md                  # who you are
├── insights/                   # agent's observations about you
├── accounts/                   # one folder per trading account
│   └── {name}/
│       ├── account.toml        # broker, type, instruments, fees
│       └── journal/            # trade logs + observations
├── memory/                     # Claude Code auto-memory
├── .claude/
│   └── skills/                 # Claude Code auto-discovers these
├── cache/                      # computed state (markdown, regenerated by sync skill)
│   ├── market.md
│   └── {group}/
│       ├── group.md
│       └── {TICKER}.md
```

On first run, `data/us-indices/` is scaffolded with SPY + QQQ as leaders and DIA + IWM as tracked. Each ticker is a directory with year-partitioned CSVs — files stay small over time. Prices are auto-fetched from Jan 1 of the current year.

### Adding a sector

Create a folder in `data/` with a `group.toml` and ticker directories:

```
~/.lafmm/data/semis/
├── group.toml
├── NVDA/
│   └── 2026.csv
├── AVGO/
│   └── 2026.csv
└── AMD/             # tracked (optional, auto-discovered)
    └── 2026.csv
```

**group.toml:**

```toml
name = "Semiconductors"
leaders = ["NVDA", "AVGO"]
swing_pct = 5.0
confirm_pct = 2.5
```

**CSV files** — OHLCV format, one row per trading day:

```csv
date,open,high,low,close,volume
2025-01-02,128.50,131.20,127.80,130.00,45123000
2025-01-03,130.10,134.50,129.60,133.00,38901000
```

The engine reads `close` for the Livermore FSM. Other columns support quant skills (ATR, volume analysis).

The filename is the ticker. If it matches a leader name, it's a leader. Any other CSV is auto-discovered as a tracked stock — gets its own engine but doesn't affect Key Price.

## The System

### The Six Columns

```
| SecRally | NatRally | UPTREND | DNTREND | NatReac | SecReac |
|  pencil  |  pencil  |  BLACK  |   RED   | pencil  | pencil  |
```

- **Upward Trend** (green) — confirmed uptrend
- **Downward Trend** (red) — confirmed downtrend
- **Natural Rally / Reaction** (gray) — significant counter-moves in either direction
- **Secondary Rally / Reaction** (gray) — indecisive moves that haven't reached prior levels

### The 18-Column Map

Livermore tracked **groups** — two leader stocks per industry:

```
| Leader A (6 cols) | Leader B (6 cols) | Key Price (6 cols) |
```

**Key Price** = A + B combined, with doubled thresholds (swing=12, confirm=6). Group trend comes from Key Price, not individual stocks.

> *"There is danger of being caught in a false movement by depending upon only one stock."*

### Signals

| Signal | Rule | Meaning |
|--------|------|---------|
| BUY (resumption) | 10(a) | Uptrend resumed through prior pivot |
| BUY (reversal) | 10(d) | Downtrend failed |
| SELL (resumption) | 10(c) | Downtrend resumed through prior pivot |
| SELL (reversal) | 10(b) | Uptrend failed |
| DANGER: Up Over | 10(e) | Rally peaked near pivot but failed, then retreated |
| DANGER: Dn Over | 10(f) | Reaction bottomed near pivot but held, then bounced |

### Market Trend

\>60% of groups bullish → market bullish. >60% bearish → market bearish. Otherwise neutral.
